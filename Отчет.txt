# Weather App

## Overview

Weather App представляет собой веб-приложение для получения актуальных данных о погоде и формирования рекомендаций по выбору одежды на основе текущих метеорологических условий. Приложение построено на базе FastAPI для бэкенда и использует ванильный JavaScript на фронтенде. Архитектура проекта разделяет логику обработки геоданных, взаимодействия с внешними погодными API и рендеринга пользовательского интерфейса.

## Core Functionality

Приложение предоставляет две основные рабочие вкладки. Первая вкладка отвечает за прямой прогноз погоды: пользователь вводит название города, после чего система определяет его географические координаты через сервис Nominatim (geopy), вычисляет локальный часовой пояс с помощью TimezoneFinder и запрашивает метеорологические показатели у Open-Meteo API. Результат включает температуру воздуха, относительную влажность, скорость ветра, количество осадков и субъективно воспринимаемую температуру.

Вторая вкладка расширяет базовый функционал, анализируя полученные погодные параметры и формируя структурированный список предметов гардероба, подходящих для текущих условий. Алгоритм учитывает температурный диапазон (отрицательные значения, прохладная погода, умеренное тепло и жара), наличие осадков и силу ветра. На основе комбинации этих факторов генерируются рекомендации по верхней одежде, нижней части гардероба, обуви и аксессуарам.

## Technical Implementation

Бэкенд приложения реализован на FastAPI с применением шаблонизатора Jinja2 для серверного рендеринга страниц. Маршрутизация сосредоточена в модуле weather.router и включает три основных эндпоинта: GET для отображения домашней страницы с сохранением последнего запрошенного города через cookies, POST для получения и обработки погодных данных по указанному городу, и GET для динамического автодополнения вводимых названий населенных пунктов.

Геокодирование выполняется через geopy с использованием Nominatim в качестве провайдера геоданных. Определение временной зоны происходит на основе географических координат с применением библиотеки timezonefinder и pytz для корректного расчета локального времени. Метеорологическая информация загружается с публичного API Open-Meteo, которое не требует ключа авторизации и предоставляет детализированные данные о текущих погодных условиях.

Фронтенд построен без использования тяжеловесных фреймворков и полагается на нативный JavaScript. Интерфейс включает механизм автодополнения с debounce-задержкой для снижения количества запросов к серверу, систему вкладок для переключения между режимами работы, а также парсинг серверного HTML-ответа для извлечения структурированных погодных данных с последующей перерисовкой в современном карточном дизайне.

## Project Structure

Корневая директория содержит точку входа в приложение и конфигурационный модуль. Логика работы с погодой вынесена в отдельный пакет app.weather, включающий роутер, схемы данных Pydantic и кастомные исключения для обработки типовых ошибок (город не найден, временная зона не определена, ошибка получения данных от внешнего API). Шаблоны страниц размещены в директории templates и представлены двумя файлами: home.html для основной разметки с вкладками и weather.html для рендеринга фрагмента с погодными показателями.

## Dependencies

Приложение опирается на следующие ключевые библиотеки: FastAPI и Uvicorn для создания и запуска веб-сервера, Jinja2 для шаблонизации, geopy и timezonefinder для работы с географическими данными и часовыми поясами, pytz для манипуляций с временными зонами, requests для выполнения HTTP-запросов к внешним API, Pydantic для валидации входных данных.

## Configuration and Execution

Настройки проекта вынесены в класс Config, который инкапсулирует параметры подключения и пути API. Запуск приложения в режиме разработки выполняется через вызов uvicorn.run с указанием хоста и порта. В production-окружении предполагается развертывание за reverse-прокси с поддержкой ASGI. CORS middleware сконфигурирован для ограничения допустимых источников запросов, что обеспечивает базовый уровень безопасности при взаимодействии с клиентской частью.

## Design Notes

Пользовательский интерфейс реализован с применением актуальных подходов к визуальному дизайну, включая эффекты стеклянного морфизма, градиентные заливки и анимированные переходы между состояниями. Адаптивная верстка обеспечивает корректное отображение как на десктопных, так и на мобильных устройствах. Состояние последнего запрошенного города сохраняется в cookies для автоматического восстановления контекста при повторном посещении страницы.
